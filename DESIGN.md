# 卡五星麻将优化设计文档 (DESIGN.md)

## 项目概况
- **技术栈**: 纯前端单 HTML 文件 (kawuxing-mahjong-backup.html)
- **游戏**: 卡五星麻将 (1v2 AI 对战)
- **牌数**: 48 张 (筒 1-9×4 + 条 1-9×4 + 字牌 中发白×4)

## 优化目标

### P0 - 修复
| 问题 | 方案 | 风险 |
|------|------|------|
| 第468行 `<script src="tiles/tile_img.js">` 引用不存在文件 | 删除该行或改为空占位 | 低风险 |

### P1 - 抢杠胡 (TDD)
**规则**: 玩家补杠时，若其他玩家能胡这张杠牌，则抢杠胡成立
**现状**: 仅对玩家(0)补杠检测了抢杠胡，AI 补杠未检测
**方案**: 
- 统一 `execAddToPung` 函数，在补杠前先检测所有玩家的胡牌可能
- TDD: 先写测试用例验证 3 种场景

### P1 - AI 防守
**现状**: AI 出牌仅考虑自身手牌效率，完全不管亮倒玩家的听牌保护
**方案**:
- 扩展 `aiPick` 函数，增加安全牌评分
- 检测所有亮倒玩家的 `waits` 列表
- 打出的牌若在对方面上，扣分惩罚

### P2 - 音效
**方案**: 使用 Web Audio API 生成音效，无需外部资源
- 摸牌: 短促"嗒"声
- 出牌: 轻微"啪"声
- 碰: 三短音
- 杠: 长音
- 胡: 胜利音效

### P2 - 视觉特效增强
- 杠牌闪光效果
- 出牌轨迹动画
- 流局提示

### P3 - 移动端适配
- 确保所有按钮触控区域 >= 44px
- 优化小屏幕布局
- 修复画布缩放 BUG
